OSI Model
7 Layers — The Foundation must know
The OSI model splits network communication into 7 abstract layers. Attacks target specific layers.

L7 Application — HTTP, DNS, FTP. XSS, SQLi, SSRF live here.
L6 Presentation — Encryption/encoding (SSL/TLS, base64).
L5 Session — Session management. Session hijacking happens here.
L4 Transport — TCP/UDP. Port scanning, SYN floods target this.
L3 Network — IP routing. ARP spoofing, IP spoofing.
L2 Data Link — MAC addresses. VLAN hopping.
L1 Physical — Cables, NICs. Physical access attacks.
TCP/IP
3-Way Handshake critical
Every TCP connection: SYN → SYN-ACK → ACK

Why it matters:
• SYN Flood — send tons of SYNs, never ACK → exhausts server
• -sS Stealth Scan — sends SYN, checks SYN-ACK, never completes
• Session Hijacking — predict sequence numbers
Key Ports
Memorize These
21FTP  22SSH  23Telnet  25SMTP  53DNS
80HTTP  443HTTPS  445SMB  3306MySQL  3389RDP

SMB (445) and RDP (3389) are historically the most exploited.
$ wireshark &
$ tcpdump -i eth0 tcp port 80
# Capture and inspect layer by layer
$ tcpdump -i eth0 'tcp[tcpflags] & tcp-syn != 0'
# See the 3-way handshake live
# Inspect ARP (L2)
$ arp -a
$ netdiscover -r 192.168.1.0/24

Linux Basics
File System & Permissions critical
Linux permission format: rwxr-xr--
r=read(4) w=write(2) x=execute(1) for [owner][group][others]

SUID bit (chmod u+s) — file runs as file owner, not caller. Massive privesc vector.
Find SUID files: find / -perm -4000 2>/dev/null

Key paths: /etc/passwd users · /etc/shadow hashes (root only) · /var/log/ logs · /tmp/ writable (privesc playground)
Kali Linux
Essential Tool Categories
Information Gathering — nmap, maltego, recon-ng
Web Application — burpsuite, sqlmap, dirb, gobuster
Password Attacks — john, hashcat, hydra
Exploitation — metasploit, msfvenom

Nmap Scan Types
Know When to Use Each exam topic
-sS (SYN Stealth) — Default, fastest, doesn't complete handshake. Requires root.
-sT (TCP Connect) — Full handshake, no root needed. Noisier, logged.
-sU (UDP) — Slower but crucial — DNS(53), SNMP(161) are UDP.
-sV — Version detection. Reveals software + version = searchable CVEs.
-O — OS detection. Tailor exploits.
-A — Aggressive (all of the above). Very noisy.
-sN/-sF/-sX — Null/FIN/Xmas. Bypass some firewalls.
NSE Scripts
Hidden Power
--script=vuln — Run all vulnerability scripts
--script=http-enum — Find web directories
--script=smb-vuln-ms17-010 — Check for EternalBlue
--script=ftp-anon — Check anonymous FTP

Scripts live in /usr/share/nmap/scripts/
# Full pentest nmap workflow
$ nmap -sV -sC -p- -T4 <target> -oN scan.txt
# Full port + version + default scripts, save output
$ nmap -p- --min-rate 5000 <target>
# Fast full port scan
$ nmap --script=vuln <target>
$ nmap -sU --top-ports 100 <target>
# UDP scan

SQL Injection
Types & Payloads critical
Classic/Error-Based — App shows SQL errors. Try: ' OR '1'='1
Union-Based — Extract other tables: ' UNION SELECT 1,table_name,3 FROM information_schema.tables--
Blind Boolean — No output: ' AND 1=1-- vs ' AND 1=2--
Time-Based — Infer from delay: '; WAITFOR DELAY '0:0:5'--

Automated: sqlmap -u "http://target/page?id=1" --dbs
Cross-Site Scripting
XSS Types & Impact bug bounty fave
Reflected XSS — Payload in URL, reflected back. Requires user to click link.
Test: <script>alert(1)</script>

Stored XSS — Saved in DB, executes for every visitor. High severity.

DOM XSS — Happens in client JS. Check innerHTML, document.write.

Real Impact: Cookie theft (document.cookie), session hijacking, keylogging, phishing overlays.

Metasploit Architecture
Key Components critical
Exploit — Code that triggers a specific vulnerability.
Payload — What runs after exploit (shell, meterpreter).
Auxiliary — Scanners, fuzzers, brute-forcers. No payload.
Post — Post-exploitation modules after you have a shell.
Encoder — Obfuscate payloads to evade AV detection.

Meterpreter = king payload. Runs in memory, encrypted comms.

Linux
Top Vectors OSCP essential
1. Sudo misconfigs — sudo -l → NOPASSWD entries
2. SUID/SGID — find / -perm -4000 → check GTFOBins
3. Cron jobs — Writable scripts run by root cron = RCE as root
4. Writable /etc/passwd — echo 'hacker::0:0::/root:/bin/bash' >> /etc/passwd

4. SeImpersonatePrivilege — Juicy/PrintSpoofer/GodPotato
5. Stored Credentials — cmdkey /list, SAM hive, WinSCP configs
6. DLL Hijacking — Place malicious DLL in search path

Run WinPEAS or PowerUp.ps1 immediately!

Malware Analysis
Static vs Dynamic
Static Analysis — Examine without running. Tools: strings, file, Ghidra, IDA Pro.
Look for: hardcoded IPs, suspicious imports (VirtualAlloc, WriteProcessMemory), packed sections.

Dynamic Analysis — Run in sandbox. Tools: Any.run, Cuckoo Sandbox, Wireshark, RegShot.
Watch for: new processes, registry writes, network connections, dropped files.
Splunk / SIEM
SOC Analyst Must-Know
SPL (Splunk Processing Language) — SQL-like for log analysis.

Failed logins: index=security EventCode=4625 | stats count by src_ip
Lateral movement: index=security EventCode=4624 LogonType=3 | rare src_ip
PowerShell detection: index=* "IEX" OR "Invoke-Expression" | table _time,host,CommandLine

Key Windows Event IDs:
4624 Logon · 4625 Failed logon · 4688 Process created
4698 Scheduled task · 7045 New service installed

EDR Evasion & Malware Dev
Modern defenses are smart. Learn to bypass them — this is where red team operators live.
👻
EDR Evasion Notes
EXPERT
EDR Bypass Techniques
Core Evasion Concepts
AMSI Bypass — Patch AMSI in memory before running PS. Classic: amsiInitFailed patch.
ETW Bypass — Blind logging by patching ntdll.dll EtwEventWrite.
Process Injection — Inject shellcode into legit processes (explorer.exe, svchost). Types: Classic, Process Hollowing, DLL Injection, Reflective DLL.
Sleep Obfuscation — Encrypt payload in memory while sleeping to evade memory scans.
Direct Syscalls — Bypass user-mode hooks by calling kernel directly (Syswhispers, Hell's Gate).

Languages: C, C++, Rust, Nim, Go preferred (Python shells get caught instantly).
